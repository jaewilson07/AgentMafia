# Reference: View interaction payloads
There are different types of payloads that can be received when users interact with [modal views](https://api.slack.com/surfaces/modals#views):
  * [`view_submission`](https://api.slack.com/reference/interaction-payloads/views#view_submission)
  * [`view_closed`](https://api.slack.com/reference/interaction-payloads/views#view_closed)


## `view_submission` payloads [](https://api.slack.com/reference/interaction-payloads/views#view_submission)[](https://api.slack.com/reference/interaction-payloads/views#view_submission)
Sent to an app's [configured Request URL](https://api.slack.com/interactivity/handling#setup) when a user submits a [view in a modal](https://api.slack.com/surfaces/modals#views).
Read [our guide to using modals](https://api.slack.com/surfaces/modals/using#handling_submissions) to learn how your app should process and respond to these payloads.
If you're using our Bolt SDKs, we have additional guidance on how to utilize the `view_submission` payload:
  * [Bolt JS](https://slack.dev/bolt-js/concepts#view-submissions)
  * [Bolt Python](https://slack.dev/bolt-python/concepts#view-submissions)
  * [Bolt Java](https://slack.dev/java-slack-sdk/guides/modals)

## `view_closed` payloads [](https://api.slack.com/reference/interaction-payloads/views#view_closed)[](https://api.slack.com/reference/interaction-payloads/views#view_closed)
Optionally sent to an app's [configured Request URL](https://api.slack.com/interactivity/handling#setup) when a user dismisses a [modal](https://api.slack.com/surfaces/modals). To receive these payloads, the modal view must [have been created](https://api.slack.com/surfaces/modals/using#composing_views) with the [`notify_on_close` argument set to `true`](https://api.slack.com/reference/surfaces/views#reference-guide-defining-view-objects__fields).
Read [our guide to using modals](https://api.slack.com/surfaces/modals/using#modal_cancellations) to learn how your app should process and respond to these payloads.
### Fields[](https://api.slack.com/reference/interaction-payloads/views#view_closed__fields)[](https://api.slack.com/reference/interaction-payloads/views#view_closed__fields)
Field | Description  
---|---  
`type` | Helps identify the source of the payload. The `type` for this interaction is `view_closed`.  
`team` | The [workspace](https://api.slack.com/methods/team.info) that the interaction happened in.  
`user` | The user who dismissed a modal to trigger this request.  
`view` | The source [modal view](https://api.slack.com/surfaces/modals#views) that the user dismissed. This will include the full state of the view, excluding any [input blocks](https://api.slack.com/reference/block-kit/blocks#input) in the modal.  
`is_cleared` | A boolean that represents whether or not a whole [view stack](https://api.slack.com/surfaces/modals#view_stack) was cleared.  
### Example[](https://api.slack.com/reference/interaction-payloads/views#view_closed__example)[](https://api.slack.com/reference/interaction-payloads/views#view_closed__example)
```
{
 "type": "view_closed",
 "team": {
  "id": "TXXXXXX",
  "domain": "coverbands"
 },
 "user": {
  "id": "UXXXXXX",
  "name": "dreamweaver"
 },
 "view": {...},
 "api_app_id": "AXXXXXX",
 "is_cleared": false
}

```
